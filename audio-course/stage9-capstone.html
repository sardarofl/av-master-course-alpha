System Capabilities

🎤 Wireless Microphone Management

8 wireless mics on ULXD system with Dante output. Frequency coordination ensures zero interference. Automatic charging stations keep batteries topped up. DSP includes individual EQ and dynamics per mic channel.

🎛️ Advanced DSP Processing

Q-SYS Core handles all processing:

  • AEC (Acoustic Echo Cancellation) for video conferencing - ceiling mics can hear remote participants through room speakers without echo
  • Automatic Gain Control (AGC) keeps presenter volume consistent
  • Feedback suppression automatically detects and notches ringing frequencies
  • Compressor/Limiter protects speakers and maintains consistent loudness
  • Parametric EQ optimized for speech intelligibility and room acoustics
  • Delay compensation time-aligns main PA with front-fill speakers

📹 Multi-Format Recording & Streaming

Recording PC captures Dante mix-minus feed (all sources except video conference return). Archives to NAS in WAV + MP4 video. Pearl-2 provides backup encoding and can stream to YouTube Live, Facebook Live, or RTMP endpoints simultaneously. Bitrate: 1080p60 @ 8Mbps.

🌐 Video Conferencing Integration

Zoom Rooms appliance connects to Dante network for audio I/O. Room audio (wireless mics + ceiling mics) → Zoom. Zoom participants → Main PA + overflow rooms. AEC processing ensures zero echo. Camera tracks active speaker automatically.

📺 Overflow Room Distribution

Video and audio from main hall distributed to overflow rooms via Dante network. Independent volume control per room. Overflow rooms can also host independent meetings with local sources when main hall not in use.

🏛️ Lobby Information System

85" display shows event schedule, countdown timers, and live video preview of main hall during events. Audio switches between background music (Sonos) when idle and live event audio during programs. Automated scheduling.

🚨 Emergency Paging Override

Rauland paging station at security desk. Press PTT button → instantly mutes ALL zones (main hall, overflow, lobby) and routes emergency announcement everywhere. When released, system returns to previous state. Fire alarm integration triggers automated evacuation messages.

🎮 Simple User Control

Presenters use Q-SYS touch panel:

  • Source select: "Laptop," "Wireless Mic 1-8," "Video Conference," "Media Player"
  • Presets: "Presentation Mode," "Town Hall," "Video Conference," "Training," "Event"
  • Volume: Main hall master, overflow rooms, lobby (independent sliders)
  • Recording: "Start Recording" / "Stop Recording" with visual confirmation
  • Emergency: Panic button calls AV support instantly

Presenter sees ONLY these controls - all complexity hidden. IT/AV staff have full system access via web UCI.

Operational Scenarios

📅 Scenario 1: Quarterly Town Hall

Setup: CEO and 3 VPs presenting to 500 employees + 200 remote viewers

System Configuration:

  1. AV tech recalls "Town Hall" preset → routes 4 wireless mics to main PA, overflow rooms, recording, and stream
  2. CEO's laptop HDMI → audio extracted → added to mix (for video playback)
  3. Recording starts automatically (scheduled in system)
  4. Streaming encoder sends to corporate intranet (RTMP)
  5. Overflow rooms show live video feed + synchronized audio
  6. Lobby display shows "Town Hall in Progress" with live preview
  7. During Q&A, employees use wireless handheld mics - auto-ducking lowers CEO mic when audience member speaks

Result: Seamless 2-hour event. Recording archived automatically. Zero technical issues.

💼 Scenario 2: Remote Executive Presentation

Setup: VP in London presenting to 300 people in main hall + overflow

System Configuration:

  1. AV tech recalls "Video Conference" preset → enables AEC on ceiling mics, routes Zoom audio to PA
  2. London VP joins Zoom meeting → appears on main screen, voice through PA system
  3. Audience members ask questions using wireless mics → mics go to Zoom (London hears them clearly)
  4. Zero echo - AEC processing subtracts PA output from ceiling mic input
  5. Recording captures both local and remote audio in sync
  6. Chat moderator monitors Zoom chat on iPad, relays questions to presenter

Result: Remote presenter feels "in the room." Audience hears both sides perfectly. Natural conversation flow.

🚨 Scenario 3: Emergency Evacuation

Setup: Fire alarm triggered during event with 500 people in main hall

System Response:

  1. Fire panel sends dry contact closure to DSP
  2. DSP immediately mutes all audio sources (presentation, music, mics)
  3. Automated voice message plays on loop: "Attention. This is an emergency. Please evacuate the building using the nearest exit. Do not use elevators."
  4. Message plays through ALL zones at maximum safe volume: main hall, overflow rooms, lobby, hallways
  5. Security can override with live paging microphone for additional instructions
  6. When fire alarm clears, DSP returns to previous state (or requires manual reset based on policy)

Result: Life safety compliance. Everyone hears evacuation instructions clearly. No confusion or missed announcements.

🎵 Scenario 4: Background Music & Idle State

Setup: No scheduled events. Building open for business.

System Configuration:

  1. Scheduled preset activates at 7:00 AM: "Background Music Mode"
  2. Sonos Port streams curated corporate playlist (Spotify)
  3. Music routed to lobby speakers only (low volume, -15dB)
  4. Main hall and overflow rooms silent (no audio)
  5. If paging station used, music ducks to -25dB, page plays, music returns
  6. At 10:00 PM, scheduled preset: "After Hours" → all audio off except emergency paging

Result: Welcoming lobby atmosphere. Zero energy wasted in unused spaces. Fully automated.

Lessons Learned & Best Practices

✅ DO: Plan for Redundancy

We specified primary and secondary network switches with automatic failover. During commissioning, we tested failover - system switched to backup path in <2 seconds with zero audio dropout. Worth every penny.

✅ DO: Invest in Quality Wireless

Shure ULXD with Dante is expensive but rock-solid. Cheap wireless systems fail during critical moments. The client didn't want a backup plan - they wanted a system that doesn't fail. ULXD delivered.

✅ DO: Proper Acoustic Measurement

We used Smaart to measure room acoustics before finalizing DSP tuning. Discovered a nasty room mode at 180Hz. Parametric EQ notch fixed it. Speech intelligibility went from "acceptable" to "excellent." Always measure!

✅ DO: Simplify User Interface

Initial control panel had 25 buttons. Presenters were intimidated. We reduced to 8 buttons: 4 presets, volume, record, help, power. User satisfaction jumped from 60% to 95%. Less is more.

❌ DON'T: Skimp on Network Infrastructure

Original spec used unmanaged switches (save $1,500). During testing, multicast storms crashed the network. We upgraded to managed switches with IGMP snooping. Cost more but saved the project.

❌ DON'T: Forget About Training

We delivered the system but didn't schedule formal training (assumed "intuitive" was enough). First event was chaos - staff didn't know where to start. We came back for 4-hour training session. Should have done it from day one.

❌ DON'T: Overlook Cable Management

Initial rack wiring was functional but messy. When we needed to troubleshoot 6 months later, it took 3× longer to trace signals. We re-did racks with proper labeling and velcro ties. Future maintenance became trivial.

❌ DON'T: Ignore Life Safety Integration

Fire marshal inspection flagged that emergency paging wasn't loud enough in main hall (during max PA volume). We had to add dedicated ceiling speakers for life safety announcements. Should have verified code compliance earlier.
