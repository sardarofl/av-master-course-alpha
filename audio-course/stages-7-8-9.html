Stage 7

Networked Audio Over IP

Encoder → Network Switch → Decoder

Audio Meets IT Infrastructure

What if instead of running hundreds of individual audio cables, you could send dozens or hundreds of audio channels over a single network cable? Welcome to Audio over IP (AoIP) - the paradigm shift that's revolutionized professional audio.

The Traditional Problem

❌ Traditional Analog/Digital Snake

  • Heavy, expensive multi-cable "snakes"
  • Limited by cable length (~300 feet max)
  • Fixed channel count (can't easily add more)
  • Analog = noise, degradation over distance
  • Nightmare to install and maintain

✅ Audio Over IP

  • Single lightweight network cable
  • 100 meters per hop, infinite with switches
  • Scalable - add devices to network easily
  • Digital = perfect quality regardless of distance
  • Uses existing IT infrastructure
  • Flexible routing - any device can talk to any other

How Audio Over IP Works

1

Encode

Audio interface converts analog audio to digital, then packages it into network packets with timing information and channel IDs.

2

Transmit

Packets are sent over standard Ethernet network (CAT5e/CAT6 cables, network switches). Uses multicast to send one stream to multiple receivers efficiently.

3

Clock Sync

All devices share a precision clock (PTP - Precision Time Protocol) so audio stays perfectly synchronized across the network. Critical for avoiding clicks and pops!

4

Receive & Decode

Receiving device subscribes to specific audio flows, decodes the packets, and converts back to analog audio or processes digitally.

Major AoIP Protocols

🎵 Dante (by Audinate)

Market leader - most popular AoIP protocol in professional audio

Key Features:

  • Plug-and-play: Automatic device discovery
  • Low latency: <1ms on gigabit networks
  • Scalable: Up to 512×512 channels per device (depending on model)
  • Redundant: Primary and secondary network paths for mission-critical applications
  • Easy routing: Dante Controller software provides simple drag-and-drop routing
  • Sample rate: 48kHz or 96kHz, up to 24-bit

Transport:

Layer 3 IP-based, uses UDP multicast, QoS-aware

Clocking:

PTP (Precision Time Protocol) for sample-accurate sync across devices

Popular Dante Devices:

  • Shure wireless mic receivers (ULXD, Axient)
  • Yamaha digital mixers (CL, QL, TF series)
  • QSC amplifiers (DPA, DCA series)
  • Focusrite RedNet interfaces
  • Symetrix DSP processors

🔄 AES67 (Interoperability Standard)

Not a protocol itself - it's an interoperability standard that allows different AoIP protocols to talk to each other.

Purpose:

Before AES67, Dante couldn't talk to Ravenna, which couldn't talk to Livewire, etc. AES67 provides a common language for cross-compatibility.

Key Features:

  • Open standard: Published by Audio Engineering Society (AES)
  • Layer 3 IP: Works over standard routed networks
  • Multicast streams: Efficient distribution
  • Sample rates: 44.1, 48, 88.2, 96 kHz
  • Resolution: Up to 24-bit linear PCM

How It Works:

Devices that support AES67 can enable an "AES67 mode" to send/receive compatible streams. Dante devices, for example, can enable AES67 to interoperate with non-Dante systems.

Important: Within a Dante network, devices use native Dante for best performance. AES67 mode is enabled only when interoperating with non-Dante devices.

Other Notable AoIP Protocols

Ravenna

High-performance protocol from ALC NetworX (now part of Lawo). Open standard, AES67 compatible. Popular in broadcast.

Livewire+ (AES67)

Telos Alliance protocol for broadcast. Simple, rock-solid reliability. Common in radio stations.

SMPTE ST 2110

Broadcast standard for video, audio, and metadata over IP. Uses AES67 for audio. For TV studios.

AVB/Milan

Audio Video Bridging - Layer 2 Ethernet protocol. Simpler than Dante but less flexible. Used by Avid, Meyer Sound.

Network Requirements for AoIP

🔴 Critical: Gigabit Network

100Mbps (Fast Ethernet) is NOT enough for serious AoIP. Use Gigabit (1000Mbps) switches minimum.

Why: 64 channels of 48kHz/24-bit audio ≈ 75 Mbps. Add control data and you're pushing 100Mbps limit.

🔴 Critical: Managed Switches

Use managed network switches with IGMP Snooping, QoS (Quality of Service), and DSCP support.

Why: Multicast audio needs IGMP snooping to route efficiently. QoS prioritizes audio over other traffic.

🟡 Important: Dedicated Network

Keep audio on a separate network (VLAN or physical) from general IT traffic.

Why: Video streams, file transfers, and web browsing can cause jitter and dropouts.

🟡 Important: Network Topology

Use star or daisy-chain topology. Avoid loops without proper Spanning Tree Protocol (STP).

✅ Recommended: Redundant Paths

For mission-critical systems, use primary and secondary network paths. Dante supports automatic failover.

✅ Recommended: Power over Ethernet (PoE)

Many AoIP devices support PoE/PoE+, eliminating separate power supplies. Cleaner installations!

Hybrid System Architecture

Integration Challenges & Solutions

⚠️ Challenge: Format Mismatches

Problem: Some sources are analog, some digital, some Dante, some HDMI-embedded

Solution: Use format converters and Dante interfaces to normalize everything into the DSP's native format (typically Dante + analog I/O)

Example: HDMI audio extractor → analog → Dante interface → DSP

⚠️ Challenge: Clocking & Sync

Problem: All digital devices need a common clock reference or you get clicks, pops, and drift

Solution: Dante network provides master clock via PTP. Analog devices don't need clocking (inherently async).

Best practice: Designate one Dante device as "preferred master" clock source

⚠️ Challenge: Control & Monitoring

Problem: System has dozens of devices - how do you control and monitor them?

Solution: Centralized control system (Crestron, AMX, Q-SYS UCI) with touchpanels. Monitor via network management software.

Dante Controller shows all Dante devices, routing, and health status

⚠️ Challenge: Latency Accumulation

Problem: Each processing stage adds latency: wireless mic (~5ms) + network (~1ms) + DSP (~5ms) = noticeable delay

Solution: Minimize processing hops. Use DSP delay compensation. For live performers, send direct monitor mixes pre-DSP.

Total system latency budget: Try to stay under 20ms for live sound

⚠️ Challenge: Priority & Paging

Problem: Emergency announcements must override everything instantly

Solution: DSP priority routing with auto-ducking. Paging mic triggers preset that mutes/ducks all music and routes page to all zones.

Fire alarm: Highest priority - mutes ALL audio, routes alarm to all speakers

⚠️ Challenge: Network Failures

Problem: If network goes down, entire audio system fails

Solution: Redundant network paths (primary and secondary switches), UPS power on network gear, local analog backup routing in DSP.

Dante redundancy: Dual network interfaces with automatic failover

System Design Workflow

  1. 1. Gather Requirements

    What sources? How many zones? What processing is needed? Budget? Any special needs (AEC for conferencing, streaming, recording)?

  2. 2. Map Signal Flow

    Draw a block diagram showing every source, processing stage, and destination. Identify format conversions needed.

  3. 3. Choose Core DSP/Matrix

    Based on I/O count, processing needs, and budget, select the brain of the system (Q-SYS, Biamp, Symetrix, etc.).

  4. 4. Plan Network Infrastructure

    How many Dante devices? Calculate bandwidth. Specify switches, cabling, redundancy. Create network diagram with VLANs if needed.

  5. 5. Select Amplifiers & Speakers

    Calculate power needs per zone. Decide on Dante amps (network flexibility) vs. analog amps (simpler, cheaper for small zones).

  6. 6. Design Control Interface

    Will users need touchpanels? Wall controllers? Mobile app? Keep it SIMPLE - users shouldn't need training.

  7. 7. Document Everything

    Create signal flow diagrams, rack elevations, network topology, DSP routing, control logic. Future you (or the next tech) will thank you!

  8. 8. Test & Commission

    Build offline, test each subsystem, integrate gradually. Measure room acoustics, tune DSP. Train end users and document final configuration.

Real-World Example: Corporate Conference Center

🏢 Fortune 500 Training Facility

The Brief:

Build a flexible conference system for a 3-building campus with 12 meeting rooms (various sizes), auditorium (300 seats), lobby, and cafeteria. Requirements: video conferencing in all rooms, campus-wide paging, background music, recording capability, easy user control.

The Solution:

Core: QSC Q-SYS Core 510i processor (128×128 I/O capacity)

Inputs:

  • 48× Shure wireless mics (handheld + lavalier) - Dante output
  • 72× Sennheiser ceiling mics (AEC for conferencing) - Dante
  • 12× HDMI audio de-embedders (presentation laptops) - Analog to Dante interfaces
  • 3× streaming service players - Dante
  • 4× paging stations (reception, security, AV control) - Analog to DSP

Network:

  • Core: 10Gb fiber backbone connecting buildings
  • Per-room: Gigabit PoE switches (Cisco 2960-X series)
  • Dedicated audio VLAN, IGMP snooping, DSCP QoS priority
  • Redundant: Primary and secondary fiber paths with SFP modules

Processing:

  • DSP per room: Parametric EQ, compressor/limiter, AEC on conferencing mics
  • Global: Emergency paging with priority ducking, campus-wide background music distribution
  • Presets: "Presentation mode," "Video Conference," "Event," "Music Only," "Emergency"

Outputs:

  • Meeting rooms: QSC CXD-Q network amplifiers (Dante in) → ceiling speakers
  • Auditorium: Line array + subwoofers (bi-amped via Dante amps)
  • Lobby/cafeteria: 70V distribution amps → ceiling speakers
  • Recording: Dante virtual soundcard → recording PC (all rooms recordable)
  • Video conferencing: Dante to Zoom Rooms appliances

Control:

  • Per-room: 7" touchpanels (volume, source select, preset recall, room combine)
  • AV staff: Q-SYS UCI web interface (full system control and monitoring)
  • Emergency: Physical paging stations override everything

Results:

✅ 100% uptime since installation (2 years)
✅ Users can operate without training ("it just works")
✅ IT staff can monitor network health remotely
✅ 40% cost savings vs. traditional per-room systems
✅ Scalable: already added 3 more rooms without major changes

🎯 Key Takeaways

  • Hybrid systems combine analog, digital, networked, and wireless technologies
  • Central DSP/matrix is the brain - handles routing, processing, control logic
  • Dante network provides flexible, scalable distribution of digital audio
  • Format converters normalize disparate sources into common format
  • Key challenges: Clocking/sync, latency management, control complexity, network reliability
  • Design workflow: Requirements → Signal flow → Core selection → Network → Amps/Speakers → Control → Documentation → Testing
  • Redundancy is critical for mission-critical installations
  • Keep control interfaces simple - users don't care about the complexity underneath
