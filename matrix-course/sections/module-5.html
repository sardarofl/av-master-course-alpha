Resolution Management

Scaling Fundamentals, EDID Strategy & Aspect Ratio Control

The Resolution Mismatch Challenge

In an ideal world, all sources would output the same resolution as all displays. In reality, you'll have 4K sources feeding 1080p displays, 1080p sources driving 4K screens, and everything in between. This module covers how to manage resolution mismatches through scaling and smart EDID management, ensuring every display shows the best possible image regardless of source resolution.

Learning Outcomes

After completing this module, you will understand:

  • When to upscale vs. downscale (and the trade-offs)
  • Aspect ratio fundamentals (4:3, 16:9, 21:9)
  • How letterbox and pillarbox work
  • EDID strategy for mixed-resolution environments
  • How to configure scaling in matrix switchers

What is Video Scaling?

A video scaler is a device or function that converts video from one resolution to another. It adjusts the pixel count to match the target display while attempting to maintain image quality.

📈 Upscaling

Process: Low resolution → High resolution

Example: 1080p → 4K

Pixel Count: 2.25x increase

The scaler interpolates new pixels based on surrounding pixels to fill the larger frame. Modern scalers use sophisticated algorithms to smooth edges and reduce artifacts.

Limitation: Upscaling cannot create detail that wasn't in the original. A 1080p source will never have 4K detail on a 4K display.

📉 Downscaling

Process: High resolution → Low resolution

Example: 4K → 1080p

Pixel Count: 4x reduction

The scaler combines or averages pixels to fit the smaller frame. This is generally higher quality than upscaling because information is removed rather than invented.

Best Practice: Always downscale when possible - it's mathematically cleaner and preserves the source's full detail at the output resolution.

Scaling Algorithms: How Scalers Work

Different scaling algorithms produce different quality results. Understanding them helps you choose the right scaler for your application.

Nearest Neighbor

Simple pixel duplication. Blocky, poor quality.

Fastest / Lowest Quality

Bilinear

Basic averaging of adjacent pixels. Smoother but blurry.

Basic / Moderate Quality

Bicubic

Uses more pixels for calculation. Sharpness preserved.

Good Quality / Standard

Lanczos

Advanced algorithm. Excellent detail preservation.

Best Quality / Professional
🎯 Professional Scalers

Professional-grade scalers (like those in Extron, Crestron, and DigitalMedia systems) use proprietary algorithms optimized for video content, not still images. They handle motion, edge enhancement, and noise reduction differently than consumer scalers.

Aspect Ratio Fundamentals

Aspect ratio is the relationship between a video's width and height, expressed as a ratio (W:H). Different formats use different aspect ratios, and converting between them requires careful handling.

Aspect Ratio Conversion Methods

When source and display aspect ratios don't match, the scaler must decide how to handle the mismatch. Common methods include:

📦 Letterbox

Use when: Wider content (2.35:1) on narrower display (16:9)

Result: Black bars added top and bottom

Example: Watching a movie on a TV - black bars above and below the image

📐 Pillarbox

Use when: Narrower content (4:3) on wider display (16:9)

Result: Black bars added left and right

Example: Watching old TV shows on a modern 16:9 TV

🔍 Zoom / Stretch

Use when: Filling the screen is more important than preserving aspect

Result: Image stretched to fill display

Warning: Stretched images look distorted - people appear wider or taller than normal

EDID Strategy for Mixed Resolution Environments

One of the most challenging scenarios is when you have mixed resolutions - some displays are 4K, some are 1080p, and they all need to work with the same sources.

⚠️ The EDID Challenge

Without scaling, if you have a 4K display and a 1080p display on the same matrix output:

  • The 4K display can show 4K content
  • The 1080p display cannot show 4K content at all
  • Result: Either the 4K display is underutilized (forced to 1080p) or the 1080p display shows nothing
📋 EDID Strategy Options
Option 1: Highest Common Denominator

Set all inputs to output 1080p EDID. All displays work, but 4K displays are underutilized. Simple but wasteful.

Option 2: Per-Output Scaling

Use a matrix with per-output scalers. Sources output at native resolution; each output scales to match its display.

Option 3: Scalable Matrix

Use a matrix with built-in scaling on inputs. Each input can be scaled to match specific outputs. Most flexible but expensive.

Option 4: 4K Matrix with Downscaling Outputs

Some matrices can output 4K to one output and 1080p to another from the same input. Best of both worlds.

Lab Exercise: Mixed Resolution Routing Scenario

Objective: Configure a system where 4K and 1080p displays must share sources, with each display showing the best possible image.

1
Scenario Setup
  • 4K laptop (source)
  • 1080p laptop (source)
  • 4K display (main room)
  • 1080p display (secondary room)
  • Matrix switcher with per-output scaling (or use external scalers)
2
EDID Configuration
  • Set 4K laptop EDID: 4K @ 60Hz (for 4K display)
  • Set 1080p laptop EDID: 1080p @ 60Hz (for 1080p display)
  • Or use matrix EDID management to copy display EDID to respective inputs
3
Routing Configuration
  • Route 4K laptop → 4K display (direct, native resolution)
  • Route 4K laptop → 1080p display (through scaler, downscale to 1080p)
  • Route 1080p laptop → 4K display (through scaler, upscale to 4K)
  • Route 1080p laptop → 1080p display (direct, native resolution)
4
Verification
  • 4K display: Shows 4K content at 4K resolution
  • 1080p display: Shows 4K content downscaled to 1080p (best quality)
  • Both laptops work on both displays

📝 Module 5 Summary

Key Takeaways:

  • Scaling Basics: Upscaling (low→high) adds pixels via interpolation; downscaling (high→low) removes pixels via averaging
  • Quality Rule: Downscaling is always higher quality than upscaling - preserve detail by reducing pixels
  • Algorithm Types: Nearest Neighbor (poor) → Bilinear (basic) → Bicubic (good) → Lanczos (best)
  • Aspect Ratios: 4:3 (1.33:1), 16:9 (1.78:1), 21:9 (2.39:1) - different formats require different handling
  • Letterbox vs Pillarbox: Letterbox = bars top/bottom (wider content); Pillarbox = bars left/right (narrower content)
  • Mixed Resolution Strategy: Use per-output scaling or downscaling matrices to serve both 4K and 1080p displays
  • EDID is Key: Set EDID to tell sources what resolution displays can accept, not what sources can output
